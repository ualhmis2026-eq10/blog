Flujo de trabajo y despliegue

Notas del equipo sobre GitHub Flow y las opciones de despliegue pedidas en la sesion.

24/03/2026 Autor: Miembro 03

El objetivo del equipo es mantener un flujo de trabajo claro: cada mejora se desarrolla en una rama propia, se revisa en GitHub y solo despues se integra en main.

En paralelo, estamos preparando el sitio para poder desplegarlo de varias formas:

  • GitHub Pages para una publicacion directa desde el repositorio,
  • Azure Static Web Apps como alternativa gestionada,
  • una maquina virtual con Nginx para completar el tercer escenario del documento.

Documentar estas decisiones en el propio blog nos sirve tambien como diario tecnico del proyecto y como evidencia de las tareas realizadas.

Practica de HMIS 2026 construida con Hugo y el tema Nightfall.
