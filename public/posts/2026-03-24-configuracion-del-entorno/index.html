Configuracion del entorno con Hugo

Pasos principales para arrancar el proyecto en local y empezar a publicar contenido.

24/03/2026 Autor: Miembro 02

Una de las ventajas de Hugo es que la puesta en marcha es rapida. Una vez instalado, basta con ejecutar el servidor de desarrollo para trabajar con recarga en caliente y comprobar los cambios al instante.

Para mantener el contenido ordenado hemos seguido una estructura basada en secciones dentro de content/:

  • content/posts/ para las entradas del blog,
  • content/equipo/ para las paginas individuales,
  • content/contacto.md para el formulario de contacto.

De esta forma el proyecto queda alineado con la forma de trabajo habitual de Hugo y resulta mas facil seguir ampliandolo durante la practica.
